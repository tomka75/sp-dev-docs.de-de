<span data-ttu-id="98c98-p102">OData verwendet URLs zum Anfordern von Informationen aus einer Ressource. Sie erstellen die URL auf eine bestimmte Weise Abfrageoptionen, die Informationen zurückgegeben, die Sie benötigen, verwenden. <span data-ttu-id="98c98-p103">In diesem Beispiel für OData-Anforderung ist so strukturiert, dass es Ruft die ersten 20 Zeilen aus einer Tabelle mit dem Namen Tabelle1 in der Arbeitsmappe ProductSales.xlsx der im Ordner "Dokumente" auf dem Server contoso1 gespeichert wird. ## <a name="service-root-uri"></a><span data-ttu-id="98c98-111">Stamm-URI-Service</span><span class="sxs-lookup"><span data-stu-id="98c98-111">Service root URI</span></span>

Sie enthält den Namen des SharePoint-Servers, auf dem die Arbeitsmappe gespeichert wird, und der Pfad, /_vti_bin/excelrest.aspx voranstellen, wie im folgenden Beispiel dargestellt.</span><span class="sxs-lookup"><span data-stu-id="98c98-p104">The initial part of the URL is called the service root and remains the same for every OData request that you make to a SharePoint server except for the name of the server. <span data-ttu-id="98c98-p105">Der dritte Teil der URL wird Abfrageoptionen für die Anforderung.
